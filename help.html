Workflow / 基本流程

1. Input / 输入

Start from text, shapes, SVG, image, fields, or random points. 先从文字、形状、SVG、图像、场或随机点开始。

2. Structure + Transform / 结构与变形

Convert, slice, repeat, distort, mix, stretch, and label data. 对数据做转换、切片、重复、扰动、混合、拉伸和标记。

3. Material + Output / 材质与输出

Apply color, distress, weathering, stack layers, preview, then export SVG or PNG. 加颜色和旧化，叠层预览后导出。

Suggested Setups / 推荐搭法

Image slicing / 图像切片

Image Input -> Grid Slice -> Shuffle Tiles or Stretch Tiles -> Preview.

图像输入后接 Grid Slice，再接 Shuffle Tiles 或 Stretch Tiles，最后 Preview。

Two-image collage / 双图混合

Two Image Inputs -> two Grid Slice nodes -> Mix Tiles -> Preview. Add a Selector to control where B appears.

两张图分别切片后进 Mix Tiles；接入 Selector 可以控制 B 图出现的位置。

Labelled slices / 带字切片

Grid Slice or Trace Slice -> Slice Labels -> Layer Stack or Image Weathering -> Preview.

切片后接 Slice Labels，再接 Layer Stack 或 Image Weathering，保留文字层。

Vector texture / 矢量纹理

Text Shape -> Distance Field -> Contour -> Ink Distress -> Preview or SVG export.

文字形状转场，再提取等高线，接 Ink Distress 后可导出 SVG。

GPT Image / OpenAI-compatible API

What you need / 需要准备

GPT Image, GPT Image Edit, Vision Judge, and Semantic Mask require an API key. Open Settings from the top toolbar, then fill API Key. If you use the official OpenAI API, leave Base URL blank; the app falls back to https://api.openai.com.

GPT Image、生图编辑、视觉判断和语义遮罩节点需要 API Key。点击顶部 Settings 填入 API Key 即可。 如果使用 OpenAI 官方 API，Base URL 可以留空；软件会默认使用 https://api.openai.com。

Third-party compatible providers / 第三方兼容服务

If your provider says it is OpenAI-compatible, paste its Base URL into Settings or into a single node's API URL field. Per-node API URL overrides the global Settings value.

如果你购买的是 OpenAI-compatible 中转服务，把服务商提供的 Base URL 填到 Settings，或者填到某个节点自己的 API URL。 节点里的 API URL 优先级高于全局设置。

Privacy / 隐私

API keys are stored only on this device and are not written into exported patch files. Generated images can be cached locally to avoid repeated cost while testing.

API Key 只保存在本机，不会写入导出的 patch 文件。生成图片可以本地缓存，方便测试时避免重复计费。

Magenta RT Music Setup / Magenta 音频模型

Why it is optional / 为什么不内置

Magenta RT models are large, so they are not included in the Nomadic Graphics installer. Mobile SAM is bundled, but Magenta should be downloaded only by users who need audio generation.

Magenta RT 模型体积较大，所以不会放进 Nomadic Graphics 安装包。Mobile SAM 已内置；Magenta 只建议需要音频生成的用户自行下载。

Apple Silicon quickstart / Mac 快速安装

The official quickstart currently recommends Python 3.12, uv, the MLX backend, then model downloads through the mrt CLI.

uv venv --python 3.12
source .venv/bin/activate
uv pip install "magenta-rt[mlx]"
mrt models init
mrt models download
mrt mlx generate --prompt "ambient pads" --duration 4.0 --model=mrt2_small

在 Mac 上建议优先用 MLX 后端。`mrt2_small` 更轻，`mrt2_base` 质量更高但更吃机器。

Windows notes / Windows 注意事项

On Windows, use the JAX backend in Nomadic Graphics. `mrt2_base` can be very slow on PC/JAX, so start with `mrt2_small` and short durations. After installing Magenta RT, open Settings and fill:

  • Magenta RT CLI, for example E:\AIEnvs\magenta-rt\Scripts\mrt.exe
  • Magenta Model/Home Folder, for example E:\AIModels\Magenta
  • Magenta Output Folder, for example E:\AIModels\Magenta\magenta-rt-v2\outputs

Windows 上建议在节点里选择 JAX。PC 跑 `mrt2_base` 可能非常慢，建议先用 `mrt2_small` 和较短时长测试。 安装好 Magenta RT 后，在 Settings 里填写 CLI、模型/Home 文件夹和输出文件夹。
